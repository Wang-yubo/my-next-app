# 系统设置功能任务清单

## 一、项目现状

### 技术栈
| 技术 | 版本 |
|------|------|
| Next.js | 16.2.3 |
| React | 19.2.4 |
| Ant Design | 6.3.5 |
| 数据库 | MongoDB + Mongoose |
| 认证 | JWT (HttpOnly Cookie) |
| 图标 | @ant-design/icons |

### 现有路由结构

```
app/
├── page.tsx                            # 登录页
├── register/page.tsx                   # 注册页
├── layout.tsx                          # 根布局 (AntdRegistry + ConfigProvider)
├── dashboard/
│   ├── layout.tsx                      # 仪表盘布局 (侧边栏 + 顶栏)
│   ├── page.tsx                        # 仪表盘首页
│   ├── loading.tsx                     # 加载骨架屏
│   ├── students/page.tsx               # 学生管理
│   ├── courses/page.tsx                # 课程管理
│   └── enrollments/page.tsx            # 选课管理
├── profile/
│   ├── layout.tsx                      # 个人资料独立布局
│   └── page.tsx                        # 个人资料页
└── api-doc/page.tsx                    # API 文档
```

### 现有认证体系
- 当前只有 `student` 和 `teacher` 两种角色
- JWT Token 存储在 HttpOnly Cookie 中
- middleware.ts 保护 /dashboard 和 /profile 路由
- 用户信息（name, role）存储在 localStorage

### 现有 Layout 体系
- **根布局** — 全局 Ant Design 配置
- **Dashboard 布局** — 侧边栏菜单 + 顶栏 + 面包屑 + 主题切换
- **Profile 布局** — 独立的简约布局

### 现状说明
"系统设置"目前只作为用户下拉菜单中的一个菜单项存在（dashboard/layout.tsx 中），**还没有实际的页面和路由**。

## 三、功能模块详细划分

### 模块一：用户管理

| 功能 | 说明 |
|------|------|
| 用户列表 | 展示所有系统用户，支持搜索、筛选、分页 |
| 创建用户 | 填写用户名、邮箱、密码、选择角色 |
| 编辑用户 | 修改用户基本信息 |
| 启用/禁用 | 控制用户账号状态 |
| 重置密码 | 管理员为指定用户重置密码 |
| 分配角色 | 为用户分配一个或多个角色 |
| 删除用户 | 软删除或硬删除 |

### 模块二：角色管理

| 功能 | 说明 |
|------|------|
| 角色列表 | 展示所有角色，如：超级管理员、教务管理员、教师、学生等 |
| 创建角色 | 定义角色名称、编码、描述 |
| 编辑角色 | 修改角色基本信息 |
| 权限分配 | **核心功能**：以树形结构勾选菜单/权限，分配到此角色 |
| 角色状态 | 启用/停用角色 |

### 模块三：菜单权限管理

| 功能 | 说明 |
|------|------|
| 菜单树管理 | 以树形结构展示所有系统菜单（支持无限层级） |
| 创建菜单项 | 定义菜单名称、图标、路由路径、排序、父级菜单 |
| 编辑菜单项 | 修改菜单配置 |
| 权限标识绑定 | 每个菜单项关联一个权限标识（如 admin:users:list） |
| 删除菜单项 | 删除时提示会影响已分配的角色 |

## 四、数据模型设计

### 1. User 模型（管理员用户表）

```
User {
  username: string       // 登录用户名 (唯一)
  password: string       // bcrypt 加密
  email: string          // 邮箱
  nickname: string       // 昵称/显示名称
  avatar: string         // 头像 URL
  status: 'active' | 'disabled'  // 账号状态
  roles: [ObjectId]      // 关联 Role 集合
  createdAt: Date
  updatedAt: Date
}
```

### 2. Role 模型（角色表）

```
Role {
  name: string           // 角色显示名称 (如"超级管理员")
  code: string           // 角色编码 (如 super_admin)
  description: string    // 角色描述
  status: 'active' | 'disabled'  // 角色状态
  permissions: [string]  // 权限标识数组 (如 ['admin:users:list', 'admin:users:create'])
  sort: number           // 排序权重
  createdAt: Date
  updatedAt: Date
}
```

### 3. Menu 模型（菜单权限表）

```
Menu {
  name: string           // 菜单显示名称
  icon: string           // 图标名称 (对应 @ant-design/icons 的组件名)
  path: string           // 路由路径
  parentId: ObjectId | null  // 父级菜单 ID (null 为顶级菜单)
  permission: string     // 权限标识 (如 'admin:users:list')
  sort: number           // 排序序号
  type: 'menu' | 'button'   // 菜单类型：菜单权限 / 按钮权限
  status: 'visible' | 'hidden' // 显示状态
  createdAt: Date
  updatedAt: Date
}
```

## 五、Layout 设计

系统管理 layout 复用 Dashboard Layout 的整体结构（侧边栏 + 顶栏 + 内容区），但替换左侧菜单为系统管理专属菜单。

### 系统管理侧边栏菜单

```
系统设置
├── 📊 系统总览 (概览统计)
├── 👥 用户管理
├── 🔐 角色管理
└── 📋 菜单权限管理
```

### 顶栏交互
- 顶栏右侧显示"返回主控台"入口，方便在 Dashboard 和 Admin 之间切换
- 保留现有用户头像下拉菜单
