# 注册模块使用说明

## 📋 功能概述

注册模块支持两种角色注册：
1. **学生** - 注册到 Students 表
2. **教师** - 注册到 Teachers 表（新建）

## 🎯 访问地址

- 登录页面: `http://localhost:3000/` 或 `http://localhost:3001/`
- 注册页面: `http://localhost:3000/register` 或 `http://localhost:3001/register`

## 📝 注册字段

### 公共字段（学生和教师都需要）
- ID（学号/工号）- 必填，唯一
- 姓名 - 必填
- 性别 - 必填（男/女）
- 年龄 - 必填
- 手机号 - 必填，需符合手机号格式
- 邮箱 - 必填，需符合邮箱格式

### 学生特有字段
- 专业 - 必填
- 年级 - 必填（例如：2024级）
- 入学日期 - 必填

### 教师特有字段
- 所属院系 - 必填
- 职称 - 必填（教授/副教授/讲师/助教）
- 学历 - 必填（博士/硕士/本科）
- 入职日期 - 必填

## 🔧 API 接口

### POST /api/register

**请求体示例（学生）：**
```json
{
  "role": "student",
  "id": "2024001",
  "name": "张三",
  "gender": "男",
  "age": 20,
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "major": "计算机科学与技术",
  "grade": "2024级",
  "enrollDate": "2024-09-01"
}
```

**请求体示例（教师）：**
```json
{
  "role": "teacher",
  "id": "T001",
  "name": "李四",
  "gender": "男",
  "age": 35,
  "phone": "13900139000",
  "email": "lisi@example.com",
  "department": "计算机学院",
  "title": "副教授",
  "education": "博士",
  "hireDate": "2020-09-01"
}
```

**成功响应：**
```json
{
  "success": true,
  "message": "注册成功",
  "data": { ... }
}
```

**失败响应：**
```json
{
  "success": false,
  "message": "错误信息"
}
```

## ✨ 页面特性

1. **科技感设计** - 与登录页保持一致的视觉风格
2. **左右布局** - 左侧欢迎信息占 2/3，右侧悬浮表单
3. **角色切换** - 点击按钮切换学生/教师注册，表单自动重置
4. **表单验证** - 完整的字段验证和错误提示
5. **重复检测** - 自动检测学号/工号和邮箱是否已存在
6. **悬浮动画** - 表单卡片有轻微的上下浮动效果
7. **返回登录** - 左上角可快速返回登录页

## 🎨 设计亮点

- 毛玻璃效果背景
- 渐变色彩方案
- 流畅的过渡动画
- 响应式交互反馈
- 统一的 placeholder 颜色
- 自定义滚动条样式

## 🚀 使用流程

1. 访问注册页面 `/register`
2. 选择注册角色（学生/教师）
3. 填写对应的表单字段
4. 点击"注册"按钮提交
5. 注册成功后自动跳转到登录页
6. 使用注册的账号登录系统

## 📌 注意事项

- 学号/工号必须唯一，不能重复注册
- 邮箱必须唯一，一个邮箱只能注册一个账户
- 手机号需要符合中国大陆手机号格式
- 注册成功后会自动跳转到登录页面
- 所有带 * 号的字段为必填项
